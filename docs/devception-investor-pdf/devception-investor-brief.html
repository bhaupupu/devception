What is Devception?02
Core Idea

Coding practice becomes a real-time social game.

Devception combines collaborative programming challenges with hidden-role gameplay. Players work together to complete a shared code problem while one or more imposters quietly try to slow the team down.

$ join lobby --language javascript --skill intermediate
role: hidden
mission: solve the codebase before time runs out
01

Game mechanics meet code

Instead of isolated submissions, developers interact through roles, tasks, sabotage, discussion, and voting.

02

Collaboration is the product

The main challenge is designed for multiple contributors, with modular fixes and shared progress.

03

Innovative learning loop

Players practice coding, communication, debugging, and judgment inside a high-engagement match format.

04

Long-term vision

How Devception Works06
Gameplay Flow

A clear match loop from lobby to win condition.

Each session is built around a fast, understandable flow that combines coding output with player behavior.

1

User joins game

Player enters the platform, chooses skill level and language, then joins a lobby.

2

Team formation

Players are grouped into a match; roles are assigned privately at start.

3

Challenge assignment

The team receives a shared modular code problem plus individual mini tasks.

4

Real-time coding

Players edit, test, chat, observe suspicious changes, and react to sabotage.

5

Progress updates

Task completion and code validation update the shared progress state.

6

Win or score

Good Coders win by solving or voting correctly; Imposters win by delaying completion.
